# Rich CLI 界面设计

## 概述

为 Weather Agent CLI 添加 Rich 驱动的终端 UI。设计参考 opencode 和 DeepSeek-Reasonix 的 CLI 前端风格。

## 核心改动

### 1. 移除独立 spinner

取消 LLM 思考期间的旋转 spinner 行。状态指示通过以下替代：

- **工具调用行**的 `⬝⬝■⬝⬝⬝` 扫描条（Knight Rider 风格），左右来回扫动
- **底部状态栏**的渐变扫描条 `▬▬▬▬▬■▬▬▬`

### 2. 配色方案

仅 3 色 + 灰色系：

| 颜色 | 用途 | Hex |
|------|------|-----|
| **蓝色** | 主色调：`❯` 前缀、工具名、扫描条、成功面板边框、状态 tag | `#58a6ff` |
| **浅蓝** | 次要高亮、LLM 回答面板标题 | `#79c0ff` |
| **红色** | 错误面板、失败状态（仅此一处） | `#f85149` |
| **浅灰** | 正文文字 | `#e6edf3` |
| **中灰** | 次要文字、标签底色 | `#8b949e` |
| **深灰** | 边框线、分隔线 | `#30363d` / `#21262d` |

### 3. 新增设计元素

| 元素 | 位置 | 实现方式 |
|------|------|----------|
| **启动横幅** | 文件顶部 | Rich Panel，显示 app 名 + 模型名（从 `config.llm_model` 读取）+ 版本 |
| **Knight Rider 扫描条** | 工具调用行 + 底部状态栏 | 自定义 Rich 动画组件，`⬝` 和 `■` 字符左右扫描 |
| **摘要脚注** | 每次回答底部 | `▣ 回答完成 · 耗时 Ns`，灰色小字 |
| **底部状态栏** | 终端末行固定 | Rich `Console` 最后一行，固定显示 |
| **双列布局** | 天气结果面板 | Rich `Columns` 或两个 `Group` 并排 |
| **LLM 回答区分** | 非天气查询结果 | 蓝色边框面板 + `💬 LLM 回答` 标题，区别于天气面板 |

### 3. 底部状态栏

```
● deepseek-chat · 已配置 · 1.2K tokens · ▬▬▬▬■▬▬▬  weather-cli v1.0.0
   ↑模型名       ↑配置状态    ↑token       ↑扫描条       ↑版本
```

- idle 状态：扫描条隐藏，显示 "idle"
- running 状态：扫描条动画 + "running" 标签
- 使用 Rich `Live` 或 `console.status` 实现实时更新

### 4. 输入框

- 使用 Rich `Prompt.ask` 或 `TextArea`（如支持）
- 边框样式：idle 时灰色 `#30363d`，focus 时蓝色 `#58a6ff`
- 前缀：青色 `❯`
- 提示小字：`按 Ctrl+D 或输入 exit 也可以退出`
- 支持 `Ctrl+C`/`Ctrl+D` 退出

### 5. debug 模式

```bash
weather-cli           # Rich UI，隐藏 LangChain verbose
weather-cli --debug   # 显示 AgentExecutor 原始日志
```

## 实现顺序

1. 添加 `rich` 到 pyproject.toml 依赖
2. 创建 `ui.py`：Console、banner、prompt、tool_call_line、weather_panel、status_bar
3. 实现扫描条动画（独立类 `Scanner`，使用 Rich `Live` + 线程）
4. 重构 `agent.py`：移除 `print`，调用 `ui.*`，处理 `--debug`
5. 测试：运行各分支（天气成功/失败/LLM闲聊/退出）
